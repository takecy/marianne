Keyboard shortcuts

Cmd is shown for macOS — on other platforms read it as Ctrl. All shortcuts are suppressed while a text-annotation overlay or any native input / textarea has focus, so the browser’s own editing keys keep working in those fields.

Open

Section titled “Open”
ShortcutAction
Cmd + OOpen an image via the File → Open… menu (native file dialog, PNG / JPEG)

Cmd + O is registered as a macOS native menu accelerator (MenuItemBuilder::with_id("file-open", ...)). The chosen image flows through the same Rust-side trust boundary used by Finder’s “Open With Marianne”, so symlink decoys and non-whitelisted extensions are rejected before any file read occurs.

Export

Section titled “Export”
ShortcutAction
Cmd + Shift + SSave the annotated PNG to a file (opens the native save dialog). Also available as File → Save As… menu accelerator (menubar display only — see verification note below)
Cmd + Shift + CCopy the annotated PNG to the system clipboard. Triple-registered as menu (File → Copy to Clipboard) + toolbar Copy button + JS keydown. Issue #28 [E] verification (PR #82) confirmed all three paths actually write to the clipboard

See Export for the full export workflow and the difference between the two paths.

Issue #28 [E] verification result (PR #82)

Manual UI verification on macOS revealed that menu-accelerator key presses (e.g. Cmd+Shift+C, Cmd+Z, Cmd+O) are handled by the JS keydown handler in WKWebView, not by the macOS menu event system. The accelerator() string on MenuItemBuilder only paints the shortcut hint in the menubar; the actual fire happens through CanvasArea.tsx’s keydown listener. Removing the JS keydowns would break keyboard shortcuts entirely. The menu route (mouse click on File → Copy to Clipboard) does work via the MenuEvent → emit("menu-action") → useMenuAction path, and preserves WKWebView’s transient user activation — clipboard writes succeed there too.

Edit history

Section titled “Edit history”
ShortcutAction
Cmd + ZUndo (up to 50 steps). Also available as Edit → Undo menu accelerator
Cmd + Shift + ZRedo. Also available as Edit → Redo menu accelerator
Delete / BackspaceDelete the selected shape (select mode only). Also available as Edit → Delete menu accelerator

History resets when you load a new image — you cannot undo back to annotations on a previous image.

Shape clipboard

Section titled “Shape clipboard”
ShortcutAction
Cmd + CCopy the selected shape to Marianne’s internal clipboard (select mode only)
Cmd + VPaste from the internal clipboard at a small offset; falls back to the OS image-paste flow when the internal clipboard is empty

Why two `Cmd + C`s?

Plain Cmd + C operates on Marianne’s internal shape clipboard — it never touches the OS clipboard. Cmd + Shift + C is the unrelated export path that puts the annotated image on the system clipboard for pasting into Slack, chat apps, or other tools.

Option + drag to duplicate

Section titled “Option + drag to duplicate”
ShortcutAction
Option (Alt) + dragDuplicate the selected shape — the original stays put, a clone is placed at the drop point (select mode only). Fires only when Option is held at both the start and the end of the drag

For the full behavior and the FigJam origin story, see Features → Option + drag to duplicate.

Tool switching

Section titled “Tool switching”
ShortcutTool
vSelect
aArrow
rRectangle
tText
mMosaic
xCrop

Tool shortcuts are modifier-less single keys (no Cmd / Ctrl / Shift / Alt). They are ignored when no image is loaded, which keeps v etc. from intercepting key presses on the empty start screen.

Crop mode

Section titled “Crop mode”

While the crop tool is active, two additional keys handle the selection:

ShortcutAction
EnterApply the crop (same as the Apply button)
EscapeCancel and return to the select tool (same as Cancel)

See Features → Image crop for the behaviour and history-reset semantics.

Discoverability tips

Section titled “Discoverability tips”
  • Hover any toolbar button to see its shortcut in the tooltip.
  • The status bar near the bottom shows the active tool icon for quick orientation.
  • Need to remember just three? r for rectangle, Cmd + Z to undo, Cmd + Shift + S to save. Everything else builds on those.
