キーボードショートカット

Cmd は macOS 表記 — 他プラットフォームでは Ctrl に読み替える。テキストアノテーションの編集オーバーレイ表示中、およびネイティブ input / textarea にフォーカスがある間は、ブラウザ標準の編集キーを尊重するため全ショートカットが無効化される。

開く

Section titled “開く”
ショートカット動作
Cmd + OFile → Open… メニューから画像を開く (ネイティブファイルダイアログ、PNG / JPEG)

Cmd + O は macOS native メニュー accelerator (MenuItemBuilder::with_id("file-open", ...)) として登録される。選択された画像は Finder の「このアプリケーションで開く」と同じ Rust 側の trust boundary を通り、symlink decoy や非 whitelist 拡張子はファイル読み取り前に拒否される。

エクスポート

Section titled “エクスポート”
ショートカット動作
Cmd + Shift + S注釈付き PNG をファイル保存 (ネイティブ保存ダイアログ)。File → Save As… メニュー accelerator にも併設
Cmd + Shift + C注釈付き PNG をシステムクリップボードへコピー。メニュー (File → Copy to Clipboard、best-effort) + toolbar Copy ボタン (信頼経路) + JS keydown (信頼経路) の三重登録。App.tsx の共有 de-dupe 層で二重発火を防止

詳しいワークフローと 2 つの経路の違いは エクスポート を参照。

編集履歴

Section titled “編集履歴”
ショートカット動作
Cmd + Zundo (最大 50 ステップ)。Edit → Undo メニュー accelerator にも併設
Cmd + Shift + Zredo。Edit → Redo メニュー accelerator にも併設
Delete / Backspace選択中シェイプを削除 (select モードのみ)。Edit → Delete メニュー accelerator にも併設

新しい画像を読み込むと履歴はリセットされる — 前画像の注釈には undo で戻れない。

シェイプクリップボード

Section titled “シェイプクリップボード”
ショートカット動作
Cmd + C選択中シェイプを Marianne の内部クリップボードへコピー (select モードのみ)
Cmd + V内部クリップボードのシェイプを少しずらしてペースト。空のときは OS の画像ペーストにフォールバック

`Cmd + C` が 2 種類ある理由

プレーンな Cmd + C は Marianne の シェイプ 専用の内部クリップボードを 操作する — OS クリップボードには触れない。Cmd + Shift + C は別経路の エクスポート機能で、注釈付き 画像 をシステムクリップボードに載せて Slack やチャットアプリへペーストできるようにする。

ツール切替

Section titled “ツール切替”
ショートカットツール
v選択 (Select)
a矢印 (Arrow)
r矩形 (Rectangle)
tテキスト (Text)
mモザイク (Mosaic)
xクロップ (Crop)

ツール切替は 修飾キーなしの単一キー (Cmd / Ctrl / Shift / Alt を含まない)。画像未読み込み時は無視される — これにより画像読み込み前のスタート画面で v 等が誤発火しない。

クロップモード

Section titled “クロップモード”

クロップツールが選択されている間は、追加で 2 つのキーが選択範囲を操作する:

ショートカット動作
Enterクロップを確定 (適用 ボタンと同じ)
Escapeキャンセルして select モードへ戻る (中止 ボタンと同じ)

挙動と履歴リセットの仕様は 機能カタログ → 画像のクロップ を参照。

ヒント

Section titled “ヒント”
  • ツールバーのボタンにホバーすると tooltip にショートカットが表示される。
  • ステータスバー (下端) には現在のツールアイコンが表示され、選択中のツールを一目で確認できる。
  • 3 つだけ覚えるなら: r で矩形、Cmd + Z で undo、Cmd + Shift + S で保存。残りはこの 3 つの応用。
