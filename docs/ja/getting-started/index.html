はじめに

このガイドでは Marianne のローカル開発環境を構築する。完了すると Tauri デスクトップアプリが起動し、テストが通り、ドキュメントサイトもローカルでプレビューできる状態になる。

技術スタック

Section titled “技術スタック”
分野採用ツール補足
デスクトップフレームワークTauri v2Rust ベース。アプリの表側はフロントエンドのみで完結
フロントエンドReact 19 + TypeScript + Vite—
描画エンジンreact-konvaKonva.js の React ラッパー
状態管理Zustandidentity-preserving な undo / redo (src/store/canvasStore.ts)
テストVitest + jsdom + Testing Library@testing-library/react, @testing-library/jest-dom
LintESLint v10flat config
フォーマッタdeno fmtPrettier ではない
docs サイトAstro Starlight今読んでいるこのサイト。site/ から /docs/ にビルドされる

Tauri Rust バックエンドは意図的に最小化 — src-tauri/src/lib.rs には greet / take_pending_open_paths / confirm_quit / renderer_ready の 4 コマンドのみ (OS 統合用)。機能開発はほぼ全て src/ で完結する。

初回セットアップ

Section titled “初回セットアップ”

  1. リポジトリをクローンし依存をインストール

  2. Playwright Chromium をインストール (ドキュメントの Mermaid SSR 用)

    ノート

    このドキュメントサイトのアーキテクチャ図はビルド時に headless Chromium で SVG に変換される。マシンあたり一度実行するだけでよく、ブラウザは ~/Library/Caches/ms-playwright (macOS) にキャッシュされ、以降の pnpm docs:build で再利用される。

  3. ホットリロード付きでデスクトップアプリを起動

    Vite dev server は http://localhost:1420 で起動する。Tauri が自動で ネイティブウィンドウでラップする。

検証コマンド

Section titled “検証コマンド”

push 前にローカルで以下を実行する。CI も同じセットを順次実行し、最初に失敗したステップで止まる。

コマンド内容
pnpm typecheckTypeScript strict チェック (tsc --noEmit)
pnpm lintESLint v10 flat config
pnpm fmt:check全ソース + Markdown に対する deno fmt --check
pnpm test:runVitest シングルラン
pnpm buildReact フロントエンドの Vite ビルド
pnpm docs:buildこのドキュメントサイトを /docs/ にビルド

このドキュメントサイトを編集する

Section titled “このドキュメントサイトを編集する”

コミット前に必ずビルドする

デプロイモデルは GitHub Pages → Branch: main / Folder: /docs。CI は コミット済み /docs/ と pnpm docs:build 出力に差分があると PR を reject する。site/ のソース編集と同時に pnpm docs:build を実行し、結果も コミットすること。

Pull Request を送る

Section titled “Pull Request を送る”
  1. worktree で feature ブランチを作成する (上記参照)
  2. 変更を実装し、すべての検証コマンドを通す
  3. Conventional Commits 形式の英語でコミットする (例: feat(canvas): add arrow rotation)
  4. デフォルトで draft モードで push & PR 作成する

リリースは .github/workflows/tagging-release.yaml で完全自動化されている。メンテナー向けの手順はリポジトリ README を参照。
