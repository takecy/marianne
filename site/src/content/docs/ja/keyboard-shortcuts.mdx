`Cmd` は macOS 表記 — 他プラットフォームでは `Ctrl` に読み替える。テキストアノテーションの編集オーバーレイ表示中、およびネイティブ `input` / `textarea` にフォーカスがある間は、ブラウザ標準の編集キーを尊重するため全ショートカットが無効化される。

## 開く

| ショートカット | 動作 |
|----------------|------|
| `Cmd + O` | **File → Open...** メニューから画像を開く (ネイティブファイルダイアログ、PNG / JPEG) |

`Cmd + O` は macOS native メニュー accelerator (`MenuItemBuilder::with_id("file-open", ...)`) として登録される。選択された画像は Finder の「このアプリケーションで開く」と同じ Rust 側の trust boundary を通り、symlink decoy や非 whitelist 拡張子はファイル読み取り前に拒否される。

## エクスポート

| ショートカット | 動作 |
|----------------|------|
| `Cmd + Shift + S` | 注釈付き PNG をファイル保存 (ネイティブ保存ダイアログ)。**File → Save As...** メニュー accelerator にも併設 (menubar 表示のみ、実発火は JS keydown、下記検証ノート参照) |
| `Cmd + Shift + C` | 注釈付き PNG をシステムクリップボードへコピー。**メニュー (`File → Copy to Clipboard`) + toolbar Copy ボタン + JS keydown** の三重登録。issue #28 `[E]` 検証 (PR #82) で 3 経路すべてが実際にクリップボードへ書き込めることを確認済み |

詳しいワークフローと 2 つの経路の違いは [エクスポート](./export/) を参照。

<Aside type="note" title="issue #28 [E] 検証結果 (PR #82)">
macOS 実機での手動 UI 検証により、**メニュー accelerator のキー押下 (例: `Cmd+Shift+C`、`Cmd+Z`、`Cmd+O`) は WKWebView 内では macOS メニューイベント系統ではなく JS keydown ハンドラで処理されている**ことが判明。`MenuItemBuilder` の `accelerator()` 文字列は menubar にショートカットヒントを表示する効果のみで、実発火は `CanvasArea.tsx` の keydown リスナが担う。JS keydown を撤去するとキーボードショートカット自体が完全に動かなくなる。一方、メニューのマウスクリック経路 (`File → Copy to Clipboard`) は `MenuEvent → emit("menu-action") → useMenuAction` 経由で正常動作し、WKWebView の transient user activation も保持される — クリップボード書き込みも成功する。
</Aside>

## 編集履歴

| ショートカット | 動作 |
|----------------|------|
| `Cmd + Z` | undo (最大 50 ステップ)。**Edit → Undo** メニュー accelerator にも併設 |
| `Cmd + Shift + Z` | redo。**Edit → Redo** メニュー accelerator にも併設 |
| `Delete` / `Backspace` | 選択中シェイプを削除 (select モードのみ)。**Edit → Delete** メニュー accelerator にも併設 |

新しい画像を読み込むと履歴はリセットされる — 前画像の注釈には undo で戻れない。

## シェイプクリップボード

| ショートカット | 動作 |
|----------------|------|
| `Cmd + C` | 選択中シェイプを Marianne の内部クリップボードへコピー (select モードのみ) |
| `Cmd + V` | 内部クリップボードのシェイプを少しずらしてペースト。空のときは OS の画像ペーストにフォールバック |

<Aside type="note" title="`Cmd + C` が 2 種類ある理由">
プレーンな `Cmd + C` は Marianne の **シェイプ** 専用の内部クリップボードを
操作する — OS クリップボードには触れない。`Cmd + Shift + C` は別経路の
エクスポート機能で、注釈付き **画像** をシステムクリップボードに載せて
Slack やチャットアプリへペーストできるようにする。
</Aside>

## Option+drag で複製

| ショートカット | 動作 |
|----------------|------|
| `Option (Alt) + ドラッグ` | 選択中のシェイプを複製。元シェイプはその場に残り、ドロップ位置にクローンが配置される (select モードのみ)。「開始時 **AND** 終了時」に Option を押している時だけ発動 |

挙動の詳細と FigJam 由来の経緯は [機能カタログ → Option+drag で複製](./features/#optiondrag-で複製) を参照。

## ツール切替

| ショートカット | ツール |
|----------------|--------|
| `v` | 選択 (Select) |
| `a` | 矢印 (Arrow) |
| `r` | 矩形 (Rectangle) |
| `t` | テキスト (Text) |
| `m` | モザイク (Mosaic) |
| `x` | クロップ (Crop) |

ツール切替は **修飾キーなしの単一キー** (Cmd / Ctrl / Shift / Alt を含まない)。画像未読み込み時は無視される — これにより画像読み込み前のスタート画面で `v` 等が誤発火しない。

## クロップモード

クロップツールが選択されている間は、追加で 2 つのキーが選択範囲を操作する:

| ショートカット | 動作 |
|----------------|------|
| `Enter` | クロップを確定 (**適用** ボタンと同じ) |
| `Escape` | キャンセルして select モードへ戻る (**中止** ボタンと同じ) |

挙動と履歴リセットの仕様は [機能カタログ → 画像のクロップ](./features/#画像のクロップ) を参照。

## ヒント

- ツールバーのボタンにホバーすると tooltip にショートカットが表示される。
- ステータスバー (下端) には現在のツールアイコンが表示され、選択中のツールを一目で確認できる。
- 3 つだけ覚えるなら: **`r` で矩形**、**`Cmd + Z`** で undo、**`Cmd + Shift + S`** で保存。残りはこの 3 つの応用。
