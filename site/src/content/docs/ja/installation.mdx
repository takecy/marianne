Marianne は **Apple Silicon Mac** 向けに dmg として配布されている。リリース dmg なら 1 分以内にインストールでき、以降はアプリ内蔵の updater が自動で最新版に追従する。手元でビルドして導入することも可能。

## システム要件

| 項目 | 要件 |
|------|------|
| ハードウェア | Apple Silicon (M系) |
| macOS | 26.0 (Tahoe) 以降 |
| ネットワーク | 起動時のアップデートチェック以外、実行時には不要 |

※ 筆者の環境 (M5, Tahoe) で開発・テストされている。古い macOS での動作は未確認。

<Aside type="caution" title="Intel Mac は非対応">
配布物は `aarch64-apple-darwin` 専用ビルド。Intel Mac で起動すると
`Bad CPU type in executable` エラーで失敗する。
</Aside>

## リリース dmg からのインストール

<Steps>
1. **dmg をダウンロード**: [GitHub Releases](https://github.com/takecy/marianne/releases) から最新の `Marianne_<version>_aarch64.dmg` を取得する。

2. **dmg をマウント**: Finder でダブルクリックする。

3. **`Marianne.app` を `/Applications` にドラッグ**: dmg ウィンドウ内の Applications ショートカット上にドロップする。

4. **dmg をアンマウント**: Finder サイドバーから取り出してクリーンアップする。

5. **Marianne を起動**: Launchpad または Spotlight から起動する。
</Steps>

### 初回起動時の Gatekeeper 警告

本 dmg は **コード署名なし** (Apple 開発者プログラム $99/年 を未払いのため) で、初回起動時に macOS が「Marianne は開発元を確認できないため開けません」と警告する場合がある。以下のいずれかを一度実行すれば解決する:

- Finder で `/Applications/Marianne.app` を右クリックし、コンテキストメニューから **開く** を選ぶ。
- 「システム設定」→「プライバシーとセキュリティ」から承認する。
- またはターミナルで以下を一度実行する:

  ```sh
  xattr -dr com.apple.quarantine /Applications/Marianne.app
  ```

一度承認すれば、以降の起動は通常通り進む。

## ソースコードからのインストール

公式バイナリを使わず、ローカルでビルドして `/Applications` に配置する方法。前提ツールは macOS 向け Tauri v2 の公式要件に準拠する。

### 前提ツール

- **Xcode Command Line Tools**: `xcode-select --install`
- **Rust**: [公式サイト](https://www.rust-lang.org/tools/install) のインストーラを使う
- **pnpm**: [公式サイト](https://pnpm.io/installation) のインストーラを使う

### ビルド手順

<Steps>
1. **リポジトリをクローン**: `git clone https://github.com/takecy/marianne.git`

2. **依存関係をインストール**: `pnpm install --frozen-lockfile`

3. **ローカルビルドしてインストール**: `pnpm install:local:unsigned` を実行する。内部で `pnpm tauri build` が走り、成果物が `/Applications/Marianne.app` にコピーされる。

4. **初回起動時の Gatekeeper 警告を承認する**: 上の「初回起動時の Gatekeeper 警告」と同じ手順で承認する。
</Steps>

<Aside type="caution" title="ソースビルド版は自動アップデート無効">
このビルドは updater 設定が無効化されている。新しいバージョンに更新するには
`git pull && pnpm install:local:unsigned` を再実行する必要がある。
</Aside>

## 自動アップデート (リリース dmg のみ)

新しい dmg を手動でダウンロードする必要はない — Marianne は起動時にリリースをチェックし、自分自身を更新する。

- 起動するたびに GitHub Releases に新しいビルドがないか問い合わせる。新版が見つかるとモーダルで通知される (バージョン番号とリリースノート付き)。
- ツールバーの **更新を確認** ボタンから手動でチェックすることもできる。
- **今すぐ更新** を選ぶとダウンロード → 自動インストール → アプリ再起動が走る。

<Aside type="caution" title="更新前に保存する">
キャンバス上の注釈はアップデートの再起動で失われる。未保存のものがある場合は
先に保存すること。注釈が存在する状態でアップデートをかけると、更新モーダルが
警告を表示する。
</Aside>

アップデートチェックが失敗した場合 (ネットワーク不通 / 署名検証失敗など) は、**更新を確認** ボタンの横に小さく `⚠ Failed` 表示が出る。アプリは通常通り動作する。ホバーで詳細メッセージが見られ、同じボタンを押せば再試行できる。

更新フィードは `https://github.com/takecy/marianne/releases/latest/download/latest.json` 固定。GitHub 上の draft / pre-release は無視され、公開済みの最新リリースのみがロールアウトされる。
