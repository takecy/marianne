Marianne は意図的にアノテーション表面を小さく保っている。4 種類のシェイプと画像クロップツール、8 色、4 段階の太さ、そして数個のジェスチャー — これでスクリーンショットのレビュー作業はほぼ全て対応できる。

## アノテーションツール

<CardGrid>
  <Card title="矩形 (Rectangle)" icon="seti:default">
    枠線のみの矩形で領域を囲む。「ここを見て」という指示に最適。線の太さは変更可能 (下記参照)。
  </Card>
  <Card title="矢印 (Arrow)" icon="right-arrow">
    Skitch 風の太い縁取り + ドロップシャドウ付き矢印を描画する。視認性が高く、複雑な背景の上でも読みやすい。配置後に両端をドラッグして向きや長さを変更可能。
  </Card>
  <Card title="テキスト (Text)" icon="document">
    キャンバス上の任意の位置にテキストラベルを配置する。四隅のリサイズハンドルでフォントサイズを変更でき、ダブルクリックで再編集できる。
  </Card>
  <Card title="モザイク (Mosaic)" icon="approve-check">
    顔や名前など機密情報を矩形領域でピクセル化する。エクスポート時には PNG に焼き付けられ、元のピクセルは復元できない。
  </Card>

ツールは左サイドバーから選択するか、単一キーショートカット (`v` / `a` / `r` / `t` / `m` / `x`) で切り替えられる — [キーボードショートカット](./keyboard-shortcuts/) を参照。

## 画像のクロップ

クロップツール (サイドバーアイコンまたはショートカット `x`) は、読み込まれている画像を切り抜き済みのものに置き換える。起動時に画像中央 80% を覆うデフォルト矩形が表示され、8 方向のリサイズハンドルが付く。矩形の本体をドラッグすれば移動、ハンドルをドラッグすればリサイズできる。**Enter** または **適用** ボタンで確定、**Escape** または **中止** ボタンで解除。

<Aside type="caution">
クロップは新しい画像を読み込むのと同じ扱いで、確定時に **undo / redo の履歴がリセットされる**。クロップ前のキャンバスに undo で戻る手段はないため、必要なら先に保存しておくこと。クロップ範囲外に完全にあるアノテーションは削除され、矩形・モザイク・矢印で範囲をまたぐものは新しい画像境界にクランプされる (矩形を一切通らない矢印は辺上に偽の矢印を生成せず、削除される)。
</Aside>

## 矩形の線の太さ

矩形は 4 段階のプリセットから線の太さを選べる。デフォルトは `thick`、つまり旧来の固定太さと同じ。

| プリセット | 太さ (px) |
|-----------|-----------|
| thin | 6 |
| medium | 12 |
| thick | 18 (デフォルト) |
| extraThick | 28 |

矢印は固定の太さプロファイル。モザイクは固定のピクセルサイズで、典型的なスクリーンショット解像度での塗りつぶしに最適化されている。

## 使い心地向上機能

### Option+drag で複製

シェイプを選択して、**Option (Alt) を押しながらドラッグする**。元シェイプはその場に残り、複製がカーソルに追従し、マウスを離すとドロップ位置に配置される。選択状態は自動で新しい複製へ移る。

<Aside type="tip">
このジェスチャーは「ドラッグ開始時 **AND** 終了時」に Option が押されている時のみ発動する。
途中で Option を離すと通常の移動として扱われる — 気が変わった時に便利。
</Aside>

### アプリ内シェイプクリップボード

シェイプを選択した状態で `Cmd/Ctrl + C` を押すと、Marianne の **内部** クリップボードにコピーされる。`Cmd/Ctrl + V` で元の位置から少しずれた場所にペーストされ、連続ペーストすると階段状に並ぶ。

内部クリップボードは OS クリップボードと完全に独立している:

- OS クリップボードに画像を持ったまま、Marianne 内でシェイプの copy/paste ができる (新規画像のペースト元として OS クリップボードを温存できる)。
- 内部クリップボードが空の場合、`Cmd/Ctrl + V` は OS の画像ペーストにフォールバックする — [画像の取り込み](./image-input/) を参照。

### Undo / Redo

画像ごとに **最大 50 ステップ** の履歴が保持される。新しい画像を読み込むと履歴はリセットされる (前画像の注釈に undo で戻ることはできない)。

## ステータスバー

ウィンドウ下端のステータスバーには以下が表示される:

- **左側**: 現在の画像の取り込み元 — フルファイルパス (drag-drop / Finder の Open With / File → Open... メニュー)、またはペースト由来なら `pasted from clipboard`。
- **右側**: 画像の自然サイズ (例: `1920 × 1080`)。

複数のスクリーンショットを並行で扱う際、「今どの画像を編集しているか」を一目で確認できる。
